# Task M030 #12 Stage 3 완료보고서 - 조회 전용 예약 안내와 최종 회귀 확인

GitHub Issue: https://github.com/postmelee/knu_lib/issues/12

## 단계 목적

조회 전용 상태에서 예약 행동이 들어와도 서버 예약 요청으로 이어지지 않게 차단하고, 사용자에게 비콘 인증이 필요하다는 이유를 명확히 안내한다. 전체 변경을 회귀 확인하고 최종 보고서 작성 전 상태를 정리한다.

## 산출물

| 파일 | 변경 내용 |
|------|-----------|
| `src/screens/SeatReservationScreen.tsx` | `handleReservation` 시작부에 `isBeaconViewOnly` guard 추가 |
| `src/screens/SeatReservationScreen.tsx` | 조회 전용 상태에서 좌석 선택 후 버튼을 누르면 `비콘 인증 필요` 안내 표시 |
| `src/screens/SeatReservationScreen.tsx` | 조회 전용 상태 버튼은 약한 danger 스타일로 표시하고, 서버 예약 요청은 실행하지 않도록 유지 |

## 본문 변경 정도 또는 본문 무손실 여부

좌석 선택 동작은 유지했다. 사용자는 비콘 미인증 상태에서도 좌석 위치와 사용 여부를 확인하고 배정 가능 좌석을 선택할 수 있다.

다만 조회 전용 상태에서 예약 버튼을 누르면 즉시 `비콘 인증 필요` 안내를 표시하고 반환하므로 `reserveMutation.mutate`는 호출되지 않는다. 비콘 인증 성공 상태의 기존 예약 확인 팝업, 예약 API 호출, 예약 성공 후 이동 흐름은 변경하지 않았다.

## 검증 결과

| `npx tsc --noEmit` | 통과 | 분리 worktree에 `node_modules`가 없어 원본 worktree의 설치본을 symlink로 임시 연결한 뒤 실행. 검증 후 symlink 제거 |
| 코드 검색 회귀 확인 | 통과 | 비콘 실패 팝업은 `확인`, 조회 전용 guard와 버튼 문구 확인. 남은 `router.back()`은 권한 필요/권한 확인 실패 팝업용 |
| `git diff --check` | 통과 | 공백 오류 없음 |

## 잔여 위험

- 실제 도서관 현장 비콘 실패 상황은 이번 단계에서 직접 재현하지 않았다.
- 비콘 재시도 버튼은 이번 작업 범위에서 제외했다. React Query error/reset 동작과 권한 재확인 흐름을 함께 설계해야 하므로 후속 개선으로 분리하는 편이 안전하다.
- 권한 거부 팝업은 여전히 열람실 목록으로 돌아갈 수 있다. 이번 Issue의 핵심은 비콘 인증 실패 후 좌석 조회 유지다.
