# docs/mydocs 추적 범위 정리 및 최신 현황 보고서 작성

## 목표

Hyper-Waterfall 도입 이후 Git에 포함할 문서 범위와 제외할 산출물을 결정하고, 최신 코드 기준 프로젝트 현황 보고서를 작성한다.

## 범위

- `docs/` 자료의 추적/제외 정책 결정
- APK, 캡처, MITM 로그의 보존 방식 검토
- `mydocs/report` 기준선 갱신
- 문서와 실제 코드 상태 불일치 목록 정리

## 검증

- `git status --short` 기준 문서 추적 범위가 명확해야 한다.
- 새 보고서에는 Auth, User/QR, Seat, Beacon, Loan 상태가 최신 코드 기준으로 정리되어야 한다.
