# Task #12 비콘 미인증 시 좌석 예약 화면 조회 전용 모드 제공

GitHub Issue: https://github.com/postmelee/knu_lib/issues/12

## 목적

좌석 예약 화면에서 비콘 인증이 실패하거나 비콘을 탐색하지 못하더라도 사용자가 좌석 목록과 잔여 좌석 상태를 확인할 수 있게 한다. 예약 자체는 기존 정책대로 비콘 인증이 필요한 작업으로 유지한다.

## 배경

현재 `SeatReservationScreen`은 비콘 인증 실패 시 `도서관 밖이신가요?` 팝업을 표시하고, 버튼 선택 시 `router.back()`으로 열람실 목록에 복귀한다. 이 동작은 실제 예약 제한과 조회 제한을 구분하지 못해, 도서관 밖이나 비콘 수신 불안정 상태에서도 좌석 현황만 확인하려는 사용자의 목적을 차단한다.

Issue #4에서 Android 비콘 인증 경로는 안정화되었지만, 비콘이 실패하는 상황 자체는 권한, 위치 서비스, Bluetooth 상태, 실제 비콘 수신 환경에 따라 여전히 발생할 수 있다. 이번 작업은 비콘 실패를 오류 종료가 아니라 조회 전용 상태로 표현하는 사용성 개선이다.

## 범위

포함 범위:

- 비콘 인증 실패 팝업의 확인 버튼을 화면 유지 동작으로 변경한다.
- 좌석 예약 화면에 비콘 미인증 또는 비콘 탐색 실패 상태를 조회 전용 모드로 표시한다.
- 조회 전용 모드에서 예약 버튼 문구와 disabled 상태를 예약 불가 사유와 일치시킨다.
- 좌석 선택 또는 예약 시도 흐름에서 비콘 인증이 필요하다는 안내를 제공한다.
- 기존 비콘 인증 성공 시 좌석 선택 및 예약 흐름은 유지한다.

제외 범위:

- 네이티브 비콘 스캔 모듈의 iOS/Android 구현 변경
- 서버 API 파라미터, 암호화, 좌석 파싱 로직 변경
- 열람실 목록 화면의 데이터 구조 변경
- 실제 도서관 현장 비콘 회귀 검증

## 설계 방향

비콘 인증 실패를 화면 이탈 조건이 아니라 `조회 전용 모드` 조건으로 다룬다. 좌석 데이터 조회는 `useReadingRoomSeats(roomId)` 경로를 그대로 사용하고, 비콘 인증 상태는 예약 가능 여부와 안내 문구를 결정하는 UI 상태로 제한한다.

사용자에게는 세 단계 상태를 명확히 보여준다.

1. 비콘 인증 진행 중: 위치 확인 중이며 예약 대기 상태
2. 비콘 인증 실패: 좌석 현황은 볼 수 있지만 예약은 비콘 인증 후 가능
3. 비콘 인증 성공: 기존과 동일하게 좌석 선택 및 예약 가능

예약 버튼은 비콘 실패 상태에서 비활성화하되, 화면 상단 또는 선택 영역 안내 문구로 왜 비활성화되었는지 먼저 설명한다. 좌석을 눌렀을 때마다 반복 팝업이 발생하지 않도록, 선택은 가능하게 두고 실제 예약 액션에서 안내하는 방식을 우선 검토한다.

## 검증 계획

- `npx tsc --noEmit`로 TypeScript 오류를 확인한다.
- 비콘 인증 성공 상태에서 기존 좌석 선택 및 예약 버튼 문구가 유지되는지 확인한다.
- 비콘 인증 실패 상태에서 화면이 열람실 목록으로 자동 복귀하지 않는지 확인한다.
- 비콘 인증 실패 상태에서 좌석 목록이 계속 표시되고 예약 액션이 차단되는지 확인한다.
- 실제 비콘 현장 검증은 이번 범위에서 제외하고, 필요 시 후속 작업으로 분리한다.

## 리스크

- React Query의 `autoBeacon.isError` 상태가 화면 재진입 또는 재시도 시 유지될 수 있어, 재시도 UI를 넣는 경우 query reset/refetch 동작을 명확히 설계해야 한다.
- 좌석 선택을 허용하면서 예약 버튼을 비활성화하면 일부 사용자가 선택 완료로 오해할 수 있으므로 안내 문구의 위치와 버튼 문구가 중요하다.
- 권한 거부와 비콘 탐색 실패는 사용자가 취할 수 있는 조치가 다르므로 같은 문구로 뭉개면 문제 해결성이 떨어질 수 있다.
