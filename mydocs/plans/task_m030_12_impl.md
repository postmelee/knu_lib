# Task #12 구현계획서 - 비콘 미인증 좌석 조회 전용 모드

GitHub Issue: https://github.com/postmelee/knu_lib/issues/12

## 구현 원칙

- 비콘 인증 실패는 좌석 예약 화면의 종료 조건이 아니라 예약 불가 상태로 처리한다.
- 좌석 목록 조회와 좌석 예약 가능 여부를 분리한다.
- 비콘 인증 성공 시 기존 좌석 선택, 예약 확인, 예약 성공 후 이동 흐름은 유지한다.
- 비콘 권한 확인, 네이티브 비콘 스캔, 서버 API 요청 형식은 변경하지 않는다.
- 구현 변경은 우선 `src/screens/SeatReservationScreen.tsx` 안에서 끝내고, 공통 컴포넌트 변경은 피한다.

## Stage 1 - 비콘 실패 상태 모델과 팝업 동작 보정

### 목표

비콘 인증 실패 시 열람실 목록으로 자동 복귀하지 않고 좌석 예약 화면에 머물도록 한다. 화면 내부에서 사용할 조회 전용 상태 값을 명확히 분리한다.

### 변경 범위

- `SeatReservationScreen`에 `isBeaconChecking`, `isBeaconViewOnly` 같은 파생 상태를 둔다.
- `autoBeacon.isError` 팝업 버튼을 `뒤로 가기`에서 `확인`으로 변경한다.
- 해당 팝업의 `onPress: router.back()` 동작을 제거한다.
- 기존 `autoBeacon.isError` 기반 예약 버튼 disable 조건은 Stage 2에서 문구와 함께 정리할 수 있도록 파생 상태를 사용하게 준비한다.

### 산출물

- `src/screens/SeatReservationScreen.tsx`
- `mydocs/working/task_m030_12_stage1.md`

### 검증

- `npx tsc --noEmit`
- 코드 리딩으로 `autoBeacon.isError` 발생 시 `router.back()`이 호출되지 않는지 확인

## Stage 2 - 조회 전용 안내 UI와 예약 버튼 문구 구현

### 목표

비콘 미인증 상태에서 사용자가 현재 화면이 조회 전용이라는 점과 예약 불가 사유를 화면 안에서 즉시 이해하게 한다.

### 변경 범위

- 좌석 선택 헤더 아래에 조회 전용 안내 영역을 추가한다.
- 안내 문구는 비콘 탐색 실패와 예약 제한을 함께 설명한다.
- 예약 버튼 문구를 상태별로 정리한다.
  - 비콘 인증 중: `도서관 위치 확인 중...`
  - 권한 확인 중: `권한 확인 중...`
  - 조회 전용: `비콘 인증 후 예약 가능`
  - 좌석 미선택: `좌석을 선택해주세요`
  - 좌석 선택 완료: `{좌석번호}번 좌석 예약하기`
- 버튼 disabled 조건은 예약 가능 여부와 일치하도록 `isBeaconViewOnly`를 사용한다.

### 산출물

- `src/screens/SeatReservationScreen.tsx`
- `mydocs/working/task_m030_12_stage2.md`

### 검증

- `npx tsc --noEmit`
- 비콘 성공/진행/실패 상태별 버튼 문구와 disabled 조건을 코드 기준으로 확인
- 안내 텍스트가 좌석맵 스크롤 영역을 과도하게 밀어내지 않도록 스타일 확인

## Stage 3 - 조회 전용 예약 시도 안내와 최종 회귀 확인

### 목표

조회 전용 상태에서 사용자가 예약 행동을 시도했을 때 이유와 다음 행동을 명확히 안내한다. 전체 변경을 최종 검증하고 보고서를 남긴다.

### 변경 범위

- 조회 전용 상태에서 예약 액션이 호출될 경우 비콘 인증 필요 안내를 표시한다.
- 좌석 선택 동작은 사용성 관점에서 유지하되, 선택 후 버튼 문구로 예약 제한을 다시 보여준다.
- 필요 시 조회 전용 안내에 `다시 시도` 액션을 추가할지 검토한다. 단, React Query reset/refetch 범위가 커지면 이번 작업에서는 제외하고 후속 이슈로 분리한다.
- 최종 결과보고서를 작성하고 오늘할일 상태를 갱신한다.

### 산출물

- `src/screens/SeatReservationScreen.tsx`
- `mydocs/working/task_m030_12_stage3.md`
- `mydocs/report/task_m030_12_report.md`
- `mydocs/orders/20260430.md`

### 검증

- `npx tsc --noEmit`
- 비콘 실패 팝업 버튼이 `확인`인지 확인
- 비콘 실패 후 화면 유지, 좌석 목록 표시, 예약 차단 상태를 코드 기준으로 확인
- 실제 도서관 현장 비콘 검증은 제외 범위로 최종 보고서에 명시

## 보류한 대안

- 좌석 클릭마다 팝업 표시: 반복 팝업이 사용성을 해칠 수 있어 기본안에서 제외한다.
- 비콘 재시도 버튼 즉시 추가: `useAutoBeaconAuth`의 React Query error/reset 상태 처리가 필요해 범위가 커질 수 있다. 구현 중 낮은 비용으로 처리 가능하면 Stage 3에서 검토하고, 아니면 후속 작업으로 분리한다.
- 공통 안내 배너 컴포넌트 추가: 현재 앱에 동일 패턴이 없어 이번 변경에서는 화면 내부 스타일로 처리한다.

## 승인 요청 사항

위 3단계 구현계획으로 Stage 1부터 진행해도 되는지 승인 요청한다. 승인 전에는 `src/` 소스 변경을 진행하지 않는다.
