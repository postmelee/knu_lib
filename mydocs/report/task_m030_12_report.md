# Task M030 #12 최종 결과보고서 - 비콘 미인증 좌석 조회 전용 모드

GitHub Issue: https://github.com/postmelee/knu_lib/issues/12

## 작업 요약

좌석 예약 화면에서 비콘 인증 실패가 발생해도 열람실 목록으로 강제 복귀하지 않고, 좌석 목록과 잔여 좌석을 계속 확인할 수 있게 했다. 예약은 기존 정책대로 비콘 인증이 필요한 작업으로 유지했고, 비콘 미인증 상태는 예약 버튼 문구와 예약 시도 안내로 표시했다. 재진입 시 이전 비콘 실패 캐시로 팝업이 즉시 재노출되지 않도록 하고, 새 인증 시도 실패 결과에 대해서만 최대 1회 안내한다.

## 잔여 위험과 후속 작업

- 실제 도서관 현장 비콘 실패 상황은 직접 재현하지 않았다. 이번 작업은 코드와 타입 검사 기준의 사용성 개선이다.
- 좌석맵 가시성을 우선해 상단의 별도 조회 전용 안내 영역은 제거했다. 조회 전용 상태는 버튼 문구와 예약 시도 시 안내 팝업으로 표현한다.
- 비콘 재시도 버튼은 포함하지 않았다. React Query error/reset 상태와 권한 재확인 흐름을 함께 설계하는 후속 작업으로 분리하는 편이 안전하다.
- 권한 거부 또는 권한 확인 실패 팝업의 `뒤로 가기` 동작은 유지했다. 이 동작까지 조회 전용으로 바꿀지는 별도 UX 판단이 필요하다.
